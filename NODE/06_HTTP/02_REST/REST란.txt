REST(REpresentational State Transfer) : 서버의 자원을 정의하고 자원에 대한 주소를 지정하는 방법
1. 주소는 의미를 명확히 전달하기 위해 명사로 구성됨 
2. 단순히 명사만 있으면 무슨 동작을 행하라는 것인지 알기 어려우므로 REST에서는 주소 외에도 HTTP 요청 메서드라는 것을 사용함
    - GET : 서버 자원을 가져오고자 할 때, 요청의 본문에 데이터를 넣지 않고 쿼리스트링을 사용함
    - POST : 서버에 자원을 새로 등록하고자 할 경우. 요청의 본문에 치환할 데이터를 넣어 보냄
    - PATCH : 서버 자원의 일부만 수정하고자 할 경우. 요청의 본문에 일부 수정할 데이터를 넣어 보냄
    - DELETE : 서버의 자원을 삭제하고자 할 경우. 요청의 본문에 데이터를 넣지 않음
    - OPTIONS : 요청을 하기 전에 통신 옵션을 설명하기 위해 사용
    이렇게 주소와 메서드만 보고 요청의 내용을 알아볼 수 있다는 장점이 있음
3. REST를 따르는 서버를 'RESTful하다'라고 표현함

예제의 REST 기반 서버 구조
GET / : restFront.html 파일 제공
GET /about : about.html 파일 제공
GET /users : 사용자 목록 제공
GET 기타 : 기타 정적 파일 제공
POST /users : 사용자 등록
PUT /user/사용자id : 해당 id의 사용자 수정
DELETE /users/사용자id : 해당 id의 사용자 제거
