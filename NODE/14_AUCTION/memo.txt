1. DB
    - mysql 사용 
    - npx sequelize init(기본 디렉터리 만들기)
    - 사용자 모델 (이메일, 닉네임, 비밀번호, 보유 자금) 사용자(1):경매(N)
    - 상품 모델 (상품명, 상품 사진, 시작 가격) 사용자(판매자1) : 상품(N), 사용자(구매자1) : 상품(N), 상품(1) : 경매(N)
    - 경매 모델 (입찰가, 입찰시 메시지), 경매 모델에는 UserId 컬럼과 GoodId 컬럼이 생성됨
    - npx sequelize db:create(데이터베이스 생성)

2. 서버센트 이벤트
    - 서버 시간을 받아오기 주기적으로 서버 시간을 조회하는 데는 양방향 통신이 필요하지 않기 때문
    - npm i sse(SSE 패키지 설치)
    - sse 모듈을 불러와 new SSE(익스프레스 서버)러 서버 객체를 생성하고 connection 이벤트 리스너를 연결하여 클라이언트와 연결할 때 어떤 동작을 할지 정의할 수 있음
    - 매개변수 client는 클라이언트에 메시지를 보낼 때 사용함.
    - 라우터에서 SSE를 사용하고 싶다면 app.set 메서드로 client 객체를 등록하고 req.app.get 메서드로 가져오면 됨
    - EventSource라는 객체를 지원하지 않기 때문에 IE나 엣지에서는 이용할 수 없음 (사용자가 직접 구현 가능)

3. 스케줄링
    - 경매가 생성되고 24시간이 지난 후에 낙찰자를 정하는 시스템을 구현하기 위해
    - npm i node-schedule
    - scheduleJob(실행될 시각, 해당 시각이 되었을 때 수행할 콜백 함수)
    - 스케줄링이 노드 기반으로 작동하므로 노드가 종료되면 스케줄 예약도 같이 종료된다는 단점
